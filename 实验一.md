# 实验一

本实验使用了助教提供的框架和数据集，目标是训练并评估一个基于 BERT 的 dense retriever。实验包含四组对比：原始 BERT-base、Stage 1 LoRA 训练后的 BERT、Stage 2 hard negatives 继续训练后的 BERT，以及成熟 embedding 模型 BGE-base-en-v1.5。

## 参数配置

参数配置

### 数据使用情况

| 项目                    | 设置 |
| ----------------------- | :--- |
| 训练集名称              | Stage 1: `data/dpr/biencoder-nq-train.json.gz`; Stage 2: `data/dpr/biencoder-nq-adv-hn-train.json.gz` |
| 训练样本数              | Stage 1: 50000; Stage 2: 50000 |
| 是否使用 hard negatives | Stage 1: 否; Stage 2: 是 |
| hard negative 样本数    | 50000 |
| 测试集名称              | `data/dpr/biencoder-nq-dev.json.gz` |
| 测试 query 数量         | 3000 |

训练时没有使用完整训练集，而是通过 `MAX_TRAIN_SAMPLES` 使用每个训练文件的前 50000 条样本。测试时使用 `MAX_EVAL_SAMPLES=3000`，即 dev 集中的 3000 个 query。所有对比方法都使用相同的 dev query 和相同的候选集合；评估候选集合由每条样本中的 `positive_ctxs + negative_ctxs + hard_negative_ctxs` 构成，因此这里的指标是同候选集排序效果，而不是全库检索效果。

### 模型参数

| 参数                  | 设置 |
| --------------------- | ---- |
| backbone              | `models/bert-base-uncased` |
| pooling 方法          | mean pooling |
| embedding 维度        | 768 |
| 是否 L2 normalization | 是 |
| 相似度计算方式        | dot product；由于向量已归一化，等价于 cosine similarity |
| max query length      | 64 |
| max passage length    | 256 |
| passage 输入格式      | `title + "\n" + text`；如果没有 title，则只使用 text |

本实验采用 bi-encoder 结构，query 和 passage 分别编码，但共享同一个 BERT encoder。BERT 输出后使用 attention mask 排除 padding token，对有效 token 的 hidden states 做 mean pooling，再进行 L2 normalization。训练和评估时均使用 query embedding 与 passage embedding 的点积作为排序分数。

### LoRA 参数

| 参数           | 设置 |
| -------------- | ---- |
| LoRA rank      | 16 |
| LoRA alpha     | 32 |
| LoRA dropout   | 0.05 |
| target modules | `query,value` |
| 可训练参数量   | 589,824 / 110,072,064，约 0.5359% |

LoRA adapter 加在 BERT attention 层的 `query` 和 `value` 线性模块上。训练时只更新 LoRA 参数，BERT 主体参数保持冻结。使用 LoRA 的原因是减少可训练参数量和显存开销，同时保留预训练 BERT 的语言表示能力，适合作为课程实验中的轻量 fine-tuning 方法。

### 训练参数

| 参数                        | 设置 |
| --------------------------- | ---- |
| epoch                       | 1 |
| batch size                  | 32 |
| gradient accumulation       | 1 |
| learning rate               | Stage 1: 2e-5; Stage 2: 5e-6 |
| warmup steps / warmup ratio | warmup ratio = 0.06 |
| weight decay                | 0.01 |
| optimizer                   | AdamW |
| scheduler                   | linear scheduler with warmup |
| temperature                 | 0.05 |
| precision                   | fp16 |
| random seed                 | 42 |
| GPU                         | NVIDIA GeForce RTX 4090 |

训练损失为 Multiple Negatives Ranking Loss / InfoNCE。每个 batch 中的 passage 矩阵由 positive passages 和 negative passages 拼接得到，第 `i` 个 query 的正确 passage 是第 `i` 个 positive passage。logits 由 query 与 passages 的点积除以 temperature 得到，然后用 cross entropy 训练。

两阶段训练参数如下：

| 阶段    | 训练数据       | epoch | learning rate | batch size | warmup | 说明 |
| ------- | -------------- | ----: | ------------: | ---------: | -----: | ---- |
| Stage 1 | 普通负例       | 1 | 2e-5 | 32 | 0.06 | 使用 `negative_ctxs` 进行基础 retriever 训练 |
| Stage 2 | hard negatives | 1 | 5e-6 | 32 | 0.06 | 从 Stage 1 adapter 继续训练，使用 `hard_negative_ctxs`，学习率更小以避免破坏第一阶段已学到的表示 |

Stage 1 学习率较大，主要用于让原始 BERT 适应检索任务；Stage 2 在 hard negatives 上继续训练，负例更难、训练信号更细，因此使用更小的学习率进行微调。

## 实验结果与分析

### 主结果表

| 方法                              | MRR@10 | Recall@10 | Recall@20 | Recall@30 |
| --------------------------------- | -----: | --------: | --------: | --------: |
| BERT-base + mean pooling          | 0.2949 | 0.2459 | 0.4028 | 0.5230 |
| BERT-base + LoRA                  | 0.4754 | 0.4238 | 0.5715 | 0.6751 |
| BERT-base + LoRA + hard negatives | 0.4923 | 0.4370 | 0.5880 | 0.6896 |
| BGE-base-en-v1.5                  | 0.7500 | 0.6616 | 0.7851 | 0.8453 |

从结果看，LoRA 训练显著超过原始 BERT。MRR@10 从 0.2949 提升到 0.4754，Recall@10 从 0.2459 提升到 0.4238，说明仅使用原始 BERT 做 mean pooling 并不能很好适配检索排序任务，而对比学习训练可以明显改善 query-passage 匹配能力。

加入 hard negatives 后，Stage 2 相比 Stage 1 继续提升，但提升幅度较小。MRR@10 从 0.4754 提升到 0.4923，Recall@10 从 0.4238 提升到 0.4370。这说明 hard negatives 能进一步帮助模型区分语义接近但不正确的 passage，不过在当前训练设置下，第二阶段只训练 1 个 epoch，且学习率较小，因此主要是稳健的小幅改进。

自己训练的 BERT retriever 与 BGE-base-en-v1.5 仍有明显差距。BGE 的 MRR@10 为 0.7500，比 Stage 2 高 0.2577；Recall@10 为 0.6616，比 Stage 2 高 0.2247。这说明成熟 embedding 模型经过更大规模、更充分的预训练和检索对齐，在同候选集排序上仍明显优于本实验中轻量训练的 BERT + LoRA。

提升同时体现在 MRR@10 和 Recall@k 上。Stage 1 相对原始 BERT 的 MRR@10 提升较大，说明正确 passage 的排名显著提前；Recall@20、Recall@30 也同步提升，说明模型不仅把正确答案排得更靠前，也提高了在较大 top-k 范围内命中的概率。Stage 2 的增益更偏小幅整体提升，符合 hard negative fine-tuning 的作用：让模型在相似负例中做更细粒度区分。

## 其他

本实验没有进行大规模超参数搜索，主要调优处理是采用两阶段训练：先用普通负例进行 LoRA 训练，再用 hard negatives 以较小学习率继续训练。该处理带来了稳定提升，但提升幅度不大。可能原因是 Stage 2 的 hard negatives 更贴近评估中的难例，能提供更强的排序学习信号；同时训练轮数较少、只训练 LoRA adapter，也限制了进一步提升空间。

后续如果继续优化，可以尝试增加训练样本数和 epoch、调整 batch size 与 temperature、扩大 LoRA target modules，或者引入更多 hard negative 构造方式。但需要注意，当前评估是同候选集排序，不代表全量 passage 库上的端到端检索表现。
